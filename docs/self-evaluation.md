# Autoevaluación del proyecto

## Introducción

En esta sección se realiza una autoevaluación del proyecto desarrollado siguiendo los criterios establecidos en el enunciado de la práctica.

El objetivo es analizar de forma crítica el trabajo realizado, valorando tanto los aspectos técnicos como organizativos del desarrollo.

---

# Valoración global

## Nota media estimada: 9.6 / 10

Considero que el proyecto cumple ampliamente los requisitos establecidos en la práctica, incluyendo funcionalidades obligatorias y avanzadas, documentación técnica, diagramas UML y uso correcto de herramientas de control de versiones.

Además, se ha realizado una reflexión crítica sobre el uso de inteligencia artificial durante el desarrollo, validando manualmente el código generado y adaptándolo a las necesidades del proyecto.
